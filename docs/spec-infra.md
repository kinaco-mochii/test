# インフラ仕様書

## 1. 概要
本ツールを動作させる実行環境（ハードウェア・OS・ミドルウェア・ネットワーク）の仕様を定める。

- 関連文書: [フロントエンド仕様書](./spec-frontend.md), [バックエンド仕様書](./spec-backend.md)

## 2. ハードウェア
- Mac mini（Apple Silicon, 統合メモリ 16GB）
- ストレージ: プロジェクトファイル・SQLite・Ollamaモデルファイルを格納するための空き容量を確保する（モデル＋プロジェクト数十件想定で50GB程度を目安とする）

## 3. OS / ランタイム
- macOS（最新版）
- Node.js（LTS版）: フロントエンド/バックエンドの実行に使用する
- Ollama: ローカルLLM推論サーバーとして常駐させる
  - モデル: Qwen2.5-Coder-7B-Instruct（Q4量子化, 約4.7GB）
  - 16GB統合メモリの制約上、複数モデルの同時ロードは避ける

## 4. プロセス構成
- フロントエンド（Next.js）: ローカルでビルド/起動、ポート例 3000
- バックエンド（Orchestrator API）: ポート例 4000
- Ollama: デフォルトポート 11434
- プレビュー用devサーバー: プロジェクトごとに動的割り当て（例 5000番台）

起動・常駐方法は `launchd`（macOSのサービス管理）または `pm2` などのプロセスマネージャを用い、Mac mini再起動時にも自動起動できるようにする（任意、MVPでは手動起動でも可）。

## 5. ネットワーク
- 社内LAN内からのみアクセス可能とする（外部公開は行わない）
- 認証機構は設けず、LANへの接続自体を信頼境界とする
- 必要に応じてファイアウォール等で社内LAN以外からのアクセスを遮断する

## 6. ストレージ構成（例）
```
/app-data/
  ├─ ollama-models/        # Ollamaが管理
  ├─ projects/
  │    ├─ <project-id>/
  │    │    ├─ frontend/   # React+Vite+Tailwindテンプレート＋生成コード
  │    │    └─ backend/    # Node+Express+SQLiteテンプレート＋生成コード
  └─ db/
       └─ app.sqlite       # プロジェクトメタデータ・チャット履歴
```

## 7. リソース管理方針
- 16GB統合メモリの制約により、同時に起動するプレビューdevサーバーは最小限に抑える（アイドル時は自動停止する）
- Ollamaモデルのロード状態を監視し、メモリ不足時はプレビュープロセスを停止してLLM推論を優先する等の方針をMVP後の運用で調整する

## 8. バックアップ/運用
- MVP時点では自動バックアップは不要（`projects/`と`db/app.sqlite`をコピーするだけで復元できる構成にしておく）
- ログファイルの保存先・ローテーション方針は今後検討する

## 9. 将来拡張の余地
- 複数ユーザー対応が必要になった場合: 認証導入、プレビューのコンテナ分離（Docker化）、リソーススケジューリングの追加
- より高性能なモデルが必要になった場合: クラウドGPUインスタンスでのOllamaホスティングへの切替、またはハイブリッド構成（設計判断のみクラウドモデル、コード生成はローカルモデル）を検討する
