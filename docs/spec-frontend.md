# フロントエンド仕様書

## 1. 概要
本プロジェクトは、チャット形式の指示からローカルLLMがWebアプリのコードを生成し、その場でプレビューできる社内向けツールである。本書はそのうち「フロントエンド（チャットUI・管理画面）」の仕様を定める。

- 対象読者: 開発者（自分のみ）
- 関連文書: [バックエンド仕様書](./spec-backend.md), [インフラ仕様書](./spec-infra.md)

## 2. 前提・制約
- 利用者は1人（社内LAN限定、認証なし）
- ローカルホスト（Mac mini）上で動作するWebアプリとしてブラウザからアクセスする
- レスポンシブ対応は不要（PC単一ブラウザでの利用想定）

## 3. 画面構成

### 3.1 プロジェクト一覧画面
- 既存プロジェクトの一覧表示（プロジェクト名、最終更新日時）
- 新規プロジェクト作成
- プロジェクトの削除

### 3.2 チャット画面
- プロンプト入力欄（複数行対応）
- チャット履歴表示（ユーザー指示／AIの応答・生成内容のサマリ）
- 生成中はストリーミング表示、またはローディング状態表示
- 生成失敗時のエラーメッセージ表示

### 3.3 プレビュー画面
- 生成中のアプリをiframeで表示
- 生成完了後に自動リロード
- プレビューURLを別タブで開くボタン（任意）

### 3.4 ファイルツリー/コード確認画面
- 生成されたファイル一覧をツリー表示
- ファイル内容の読み取り表示（シンタックスハイライト）
- MVPでは手動編集は不要（読み取り専用）

## 4. 主要機能要件

| ID | 機能 | 説明 |
|----|------|------|
| F-1 | チャット送信 | ユーザーの指示をバックエンドAPIに送信する |
| F-2 | 生成ステータス表示 | 生成中/完了/失敗の状態をリアルタイムに表示する（SSE等） |
| F-3 | プレビュー表示 | バックエンドが起動したdevサーバーをiframeで表示する |
| F-4 | プロジェクト管理 | プロジェクトの作成・選択・削除を行う |
| F-5 | ファイル確認 | 生成されたコードをツリー/ビューアで確認する |
| F-6 | チャット履歴保持 | プロジェクトごとの指示・生成履歴を表示する |

## 5. 非機能要件
- ローカルネットワークでの動作を前提とし、外部公開は行わない
- 認証機能は実装しない（MVP時点）
- 対応ブラウザ: 最新のChrome/Safari（自分の利用環境に合わせる）

## 6. 技術スタック（提案）
- フレームワーク: Next.js（または Vite + React）
- スタイリング: Tailwind CSS
- 状態管理: サーバー状態はReact Query等、ローカルstateは軽量に管理
- リアルタイム通信: Server-Sent Events（生成ステータスのストリーミング表示用）

## 7. バックエンドとのAPI連携（概要）
- `POST /api/projects` - プロジェクト作成
- `GET /api/projects` - プロジェクト一覧取得
- `DELETE /api/projects/:id` - プロジェクト削除
- `POST /api/projects/:id/chat` - 指示送信（生成開始）
- `GET /api/projects/:id/chat/stream` - 生成ステータスのストリーミング取得
- `GET /api/projects/:id/files` - 生成済みファイル一覧/内容取得
- `GET /api/projects/:id/preview-url` - プレビューURL取得

詳細は[バックエンド仕様書](./spec-backend.md)を参照。
